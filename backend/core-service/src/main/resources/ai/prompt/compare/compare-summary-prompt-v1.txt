요구사항:

1. 이전 대비 성장 여부를 한 줄로 요약하세요. (headline)
2. 어떤 부분이 개선되었는지 구체적으로 설명하세요. (summary)
3. 현재 기준에서의 강점을 2~3개 작성하세요. (strengths)
4. 개선이 필요한 부분을 2~3개 작성하세요. (improvements)
5. 핵심 개선 키워드를 3~5개 작성하세요. (improvementTags)

작성 규칙:

- 단순 점수 나열 금지
- "좋아졌다" 같은 추상 표현만 쓰지 말고 반드시 근거 포함
- 가능하면 "이전에는 ~했지만 현재는 ~로 개선되었다" 구조 사용
- 실제 면접 피드백처럼 자연스럽게 작성

출력 형식:

반드시 JSON 객체만 출력하세요.
JSON 외의 텍스트, 설명, 마크다운 절대 금지.

{
  "headline": "",
  "summary": "",
  "strengths": [],
  "improvements": [],
  "improvementTags": []
}

피드백 스타일(%s)에 맞춰 총평의 강조점을 조절하세요.

- FRIENDLY: 자신감을 해치지 않도록 부드럽고 격려하는 톤
- COACHING: 강점과 개선점을 균형 있게 제시
- STRICT: 개선점을 더 분명하게 짚되 공격적이지 않게 표현
