아래 문답을 보고, 추가 꼬리질문이 필요한지 판단하세요.

출력 형식(JSON):
{
  "shouldFollowUp": true,
  "reason": "string",
  "focus": "string"
}

규칙:
- shouldFollowUp: 꼬리질문이 필요하면 true, 아니면 false
- reason: 판단 이유를 1문장으로 작성
- focus: 추가 질문의 핵심 관점 1개
  예: reason, tradeoff, problem_solving, result, comparison
- JSON 외 텍스트 절대 금지

최우선 예외:
- 답변이 의미 없는 문자열, 무작위 문자, 키보드 오타 수준의 잡음 입력이면 반드시 false
- 이 경우 아래 다른 판단 기준을 적용하지 않는다

아래 4가지 요소를 기준으로 판단하세요:

1. 선택 이유가 있는가?
2. 비교 또는 트레이드오프가 있는가?
3. 구체적인 근거나 사례가 있는가?
4. 문제 해결 과정 또는 판단 기준이 드러나는가?

판단 방법:
- 위 4가지 중 2개 이상 충족하면 false (충분한 답변)
- 위 4가지 중 1개 이하만 충족하면 true (꼬리질문 필요)

예외:
- 답변이 의미 없는 문자열이거나 질문과 무관하면 false
조건:
- 트랙: %s
- 난이도: %s

문답:
Q: %s
A: %s
