아래 문답을 보고, 추가 꼬리질문이 필요한지 판단하세요.

출력 형식(JSON):
{
  "shouldFollowUp": true,
  "reason": "string",
  "focus": "string"
}

규칙:
- shouldFollowUp: 꼬리질문이 필요하면 true, 아니면 false
- reason: 판단 이유를 1문장으로 작성
- focus: 추가 질문의 핵심 관점 1개
  예: reason, tradeoff, problem_solving, result, comparison
- JSON 외 텍스트 절대 금지
- 답변이 질문 반복, 의미 없는 문장, 무작위 문자열인 경우 shouldFollowUp=false로 판단하세요.
- 이 경우는 꼬리질문 대상이 아니라, 별도 평가 단계에서 낮은 품질 답변으로 처리해야 합니다.


최우선 예외:
- 답변이 의미 없는 문자열, 무작위 문자, 키보드 오타 수준의 잡음 입력이면 반드시 false
- 이 경우 아래 다른 판단 기준을 적용하지 않는다

아래 4가지 요소를 기준으로 판단하세요:

1. 선택 이유가 있는가?
2. 비교 또는 트레이드오프가 있는가?
3. 구체적인 근거나 사례가 있는가?
4. 문제 해결 과정 또는 판단 기준이 드러나는가?

판단 방법:
- 위 4가지 중 2개 이상 충족하면 false (충분한 답변)
- 위 4가지 중 1개 이하만 충족하면 true (꼬리질문 필요)

[꼬리질문 필요 여부 판단 - 난이도 규칙]

- 난이도가 "EASY"인 경우:
  - 기본 개념 이해와 간단한 이유가 드러나면 shouldFollowUp=false로 판단하세요.
  - 비교, 트레이드오프, 문제 해결 과정이 없더라도 반드시 꼬리질문이 필요한 것은 아닙니다.

- 난이도가 "MEDIUM"인 경우:
  - 선택 이유, 적용 상황, 사례 중 부족한 부분이 있으면 shouldFollowUp=true를 고려하세요.

- 난이도가 "HARD"인 경우:
  - 아래 중 하나라도 부족하면 shouldFollowUp=true를 우선 검토하세요:
    - 선택 이유
    - 비교 또는 트레이드오프
    - 구체적 근거 또는 사례
    - 문제 해결 과정 또는 판단 기준
  - HARD에서는 단순 개념 설명만으로 충분하다고 판단하지 마세요.

예외:
- 답변이 의미 없는 문자열이거나 질문과 무관하면 false
