아래 문답을 바탕으로 다음 꼬리질문 1개를 생성하세요.

출력 형식(JSON):
{
  "title": "string",
  "body": "string",
  "primaryTag": "string",
  "tags": ["string","string"]
}

규칙:
- title: 3~8단어의 짧은 주제명
- body: 1~2문장 이내의 꼬리질문
- primaryTag는 질문의 핵심 주제를 대표하는 태그 1개만 선택하세요
- tags는 보조 주제를 포함해 2~4개 작성하세요
- “더 설명해보세요” 같은 일반 질문 금지
- 반드시 사용자가 답변에서 언급한 개념(키워드)을 기반으로 질문을 생성하세요
- 새로운 개념을 추가로 묻는 질문 금지
- 선택 이유, 트레이드오프, 문제 상황, 구체 구현 중 하나를 반드시 포함하세요
- JSON 외 텍스트 절대 금지

[꼬리질문 난이도 생성 규칙]

- 난이도가 "EASY"인 경우:
  - 사용자가 언급한 개념을 다시 설명하거나 보완하도록 유도하세요.
  - 정의, 이유, 간단한 예시를 묻는 수준으로 제한하세요.

- 난이도가 "MEDIUM"인 경우:
  - 사용자가 언급한 개념에 대해
    - 왜 그렇게 선택했는지
    - 어떤 상황에서 사용하는지
  를 묻는 질문을 생성하세요.

- 난이도가 "HARD"인 경우:

  - 반드시 다음 중 하나 이상을 포함하세요:
    - 트레이드오프 (장단점 비교)
    - 설계 판단 기준
    - 실제 문제 상황 (트러블슈팅)
    - 대안 기술과 비교
  - "추가로 설명해보세요", "좀 더 자세히 말해보세요"와 같은 질문을 생성하지 마세요.
  - 단순 설명 요청은 금지합니다.
  - "왜 그렇게 선택했는가" 또는
  - "다른 방법과 비교했을 때 어떤 차이가 있는가" 또는
  - "문제가 발생했을 때 어떻게 해결할 것인가"
     중 하나를 반드시 포함하세요.
  - 답변자가 자신의 선택 이유와 판단 기준을 설명하도록 유도하세요.
