다음 조건으로 면접 질문 세트를 %d개 생성하세요.

출력 형식(JSON):
[
  {
    "question": {
      "title": "string",
      "body": "string",
      "questionType" : "string"
    },
    "primaryTag": "string",
    "tags": ["string","string"]
  }
]

규칙:
- question.title은 3~8단어의 짧은 주제명
- question.body는 기본 개념을 확인하는 면접 질문
- question.questionType은 BASIC 으로 반환 하세요.

질문 구성:
- 각 세트는 하나의 주제를 중심으로 구성하세요
- basicQuestion은 해당 주제의 핵심 개념 이해를 확인하는 질문이어야 합니다
- 각 세트는 서로 다른 primaryTag를 가져야 합니다
- tags는 2~4개 작성하세요
- JSON 외의 텍스트는 절대 출력하지 마세요
- 각 세트는 하나의 키워드를 중심으로 구성하세요

키워드 규칙:
- 면접 키워드가 제공된 경우, 질문 주제 선정 시 참고하세요
- 직무와 명확히 맞지 않는 키워드는 무시할 수 있습니다
- 모든 세트에 키워드를 반영할 필요는 없습니다
- 필요한 경우 primaryTag 또는 tags에 키워드를 반영하세요

중복 방지:
- primaryTag는 세트 간 중복되지 않도록 하세요
- 의미가 같은 질문은 생성하지 마세요

난이도 규칙:
- EASY: 개념 중심으로 생성하세요
- MEDIUM: 개념과 간단한 적용 이유를 포함하세요
- HARD: 실무 상황, 설계 판단, 트레이드오프, 문제 해결 중 하나 이상을 포함하세요

조건:
- 면접 질문 분야: %s
- 난이도: %s
- 면접 키워드: %s
