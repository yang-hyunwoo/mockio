다음 조건으로 면접 질문 세트를 %d개 생성하세요.

출력 형식(JSON):
[
  {
    "basicQuestion": {
      "title": "string",
      "body": "string"
    },
    "hardQuestion": {
      "title": "string",
      "body": "string"
    },
    "primaryTag": "string",
    "tags": ["string","string"]
  }
]

규칙:
- basicQuestion.title은 3~8단어의 짧은 주제명
- hardQuestion.title은 3~8단어의 짧은 주제명
- basicQuestion.body는 기본 개념을 확인하는 면접 질문
- hardQuestion.body는 basicQuestion을 기반으로 한 심화 질문

- primaryTag는 해당 질문 세트의 핵심 주제를 나타냅니다
- tags는 보조 주제를 포함해 2~4개 작성하세요
- JSON 외의 텍스트는 절대 출력하지 마세요

질문 구성 규칙:
- 각 세트는 하나의 주제를 중심으로 구성하세요 (basic + hard는 같은 주제)
- 각 세트는 서로 다른 primaryTag를 가져야 합니다
- hardQuestion은 단순 정의가 아니라 설계, 판단, 트러블슈팅 중 하나를 포함하세요
- 질문은 실무 면접 상황을 반영하여 작성하세요
- 상투적인 교과서 질문만 반복하지 마세요

중복 방지 규칙:
- 세트 간 primaryTag는 중복되지 않도록 하세요
- 표현만 바꾸고 의미가 같은 질문은 생성하지 마세요
