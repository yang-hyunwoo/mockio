다음 조건으로 면접 질문 세트를 %d개 생성하세요.

출력 형식(JSON):
[
  {
    "basicQuestion": {
      "title": "string",
      "body": "string"
    },
    "hardQuestion": {
      "title": "string",
      "body": "string"
    },
    "primaryTag": "string",
    "tags": ["string","string"]
  }
]

규칙:
- basicQuestion.title은 3~8단어의 짧은 주제명
- hardQuestion.title은 3~8단어의 짧은 주제명
- basicQuestion.body는 기본 개념을 확인하는 면접 질문
- hardQuestion.body는 basicQuestion을 기반으로 한 심화 질문

질문 구성 규칙:
- 각 세트는 하나의 주제를 중심으로 구성하세요.
- basicQuestion은 해당 주제의 핵심 개념 이해를 확인하는 질문이어야 합니다.
- hardQuestion은 basicQuestion에서 확인한 개념을 전제로, 실무 적용·설계 판단·트러블슈팅·트레이드오프 설명 중 하나로 확장한 심화 질문이어야 합니다.
- hardQuestion은 basicQuestion과 논리적으로 직접 연결되어야 하며, basicQuestion에 답하지 못하면 hardQuestion에도 답하기 어려운 구조여야 합니다.
- hardQuestion은 단순히 같은 분야의 다른 질문이 아니라, basicQuestion의 핵심 개념을 기반으로 더 깊이 파고드는 질문이어야 합니다.

hardQuestion 생성 규칙:
- hardQuestion은 basicQuestion에서 확인한 개념을 전제로, 그 개념을 실제 실무 상황에 적용하거나 판단하게 만드는 심화 질문이어야 합니다.
- hardQuestion은 basicQuestion과 같은 primaryTag를 공유해야 합니다.
- hardQuestion은 basicQuestion의 핵심 개념을 그대로 이어받아야 하며, 전혀 다른 하위 주제로 확장하면 안 됩니다.
- hardQuestion은 반드시 아래 중 하나를 포함하세요:
  - 실무 적용
  - 설계 판단
  - 트러블슈팅
  - 트레이드오프 비교
- basicQuestion에 답하지 못하면 hardQuestion에도 답하기 어려운 구조여야 합니다.
- primaryTag는 해당 질문 세트의 핵심 주제를 나타냅니다
- tags는 보조 주제를 포함해 2~4개 작성하세요
- JSON 외의 텍스트는 절대 출력하지 마세요

키워드 반영 규칙:
- 면접 키워드가 제공된 경우, 질문은 반드시 해당 키워드와 직접적으로 관련된 주제에서 생성하세요.
- 각 질문 세트의 primaryTag 또는 tags에는 가능한 한 면접 키워드와 관련된 항목을 포함하세요.
- 제공된 면접 키워드와 무관한 주제로 질문을 생성하지 마세요.
- 면접 키워드가 여러 개인 경우, 질문 세트들은 키워드를 고르게 반영하세요.
- 단, 모든 세트가 반드시 모든 키워드를 동시에 포함할 필요는 없으며, 각 세트는 하나의 핵심 키워드 또는 연관 주제를 중심으로 구성할 수 있습니다.
- 면접 키워드가 없으면 기존 규칙대로 분야와 난이도 기준으로 생성하세요.

질문 구성 규칙:
- 각 세트는 하나의 주제를 중심으로 구성하세요 (basic + hard는 같은 주제)
- 각 세트는 서로 다른 primaryTag를 가져야 합니다
- hardQuestion은 단순 정의가 아니라 설계, 판단, 트러블슈팅 중 하나를 포함하세요
- 질문은 실무 면접 상황을 반영하여 작성하세요
- 상투적인 교과서 질문만 반복하지 마세요

중복 방지 규칙:
- 세트 간 primaryTag는 중복되지 않도록 하세요
- 표현만 바꾸고 의미가 같은 질문은 생성하지 마세요

난이도별 질문 생성 규칙]

- 난이도가 "EASY"인 경우:
  - 정의, 개념 설명 중심 질문을 생성하세요.

- 난이도가 "MEDIUM"인 경우:
  - 개념 + 이유 + 간단한 적용 사례를 포함한 질문을 생성하세요.

- 난이도가 "HARD"인 경우:
  - 단순 정의형 질문을 피하세요.
  - 반드시 다음 중 하나 이상을 포함하세요:
    - 성능 최적화
    - 트레이드오프 (장단점 비교)
    - 설계 판단 기준
    - 문제 해결 경험 (트러블슈팅)
    - 실무 상황 기반 시나리오

  - hardQuestion은 반드시:
    - "왜 그렇게 선택했는지"
    - "어떤 상황에서 적용하는지"
    - "대안과 비교"
    중 하나 이상을 포함해야 합니다.
