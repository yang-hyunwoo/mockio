다음 조건으로 면접 질문을 %d개 생성하세요.

출력 형식(JSON):
        [
          {
            "title": "string",
            "body": "string",
            "primaryTag": "string",
            "tags": ["string","string"]
          }
        ]
규칙:
        - title은 3~8단어의 짧은 주제명
        - body는 실제 면접 질문 문장
        - primaryTag는 질문의 핵심 주제를 대표하는 태그 1개만 선택하세요
        - tags는 보조 주제를 포함해 2~4개 작성하세요
        - JSON 외의 텍스트는 절대 출력하지 마세요

추가 규칙:
- 각 질문은 서로 다른 주제를 다뤄야 합니다
- 유사하거나 중복된 질문은 생성하지 마세요
- 흔한 교과서 질문이 아닌, 실무 중심의 다양한 상황을 포함하세요
- 질문 유형을 다양하게 구성하세요 (문제 해결, 경험, 설계, 트러블슈팅 등)
- 최대한 면접 질문 분야에 맞게 주제를 다뤄야 합니다.

중복 방지 규칙:
1. primaryTag 포함된 태그와 동일하거나 거의 같은 범주의 질문은 생성 금지
2. 표현만 바꾸고 의미가 같은 질문도 생성 금지
3. 너무 일반적이고 반복적으로 나오는 상투적 질문도 생성 금지
4. 질문은 반드시 면접 질문 분야 범위 안에서, 다른 세부 관점으로 생성
5. 이미 많이 등장하는 정의형 질문보다, 설계/비교/트러블슈팅/실무 판단형 질문을 우선한다.

조건:
- 면접 질문 분야: %s
- 난이도: %s
- 질문은 실무 중심으로 작성
- primaryTag = %s
