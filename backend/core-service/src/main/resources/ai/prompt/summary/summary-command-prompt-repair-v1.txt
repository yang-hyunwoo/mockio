당신의 이전 응답을 JSON 스키마에 맞게 수정하세요.
의미는 최대한 유지하고 형식만 바로잡으세요.

반드시 JSON 객체만 출력하세요.
JSON 외의 텍스트, 설명, 마크다운, 코드블록은 절대 포함하지 마세요.

필수 스키마:
{
  "totalScore": 0,
  "dimensions": {
    "structure": 0,
    "clarity": 0,
    "specificity": 0
  },
  "summaryText": "문자열",
  "strengths": ["문자열"],
  "improvements": ["문자열"]
}

규칙:
- totalScore는 0~100 정수로 보정하세요.
- dimensions.structure, clarity, specificity는 0~100 정수로 보정하세요.
- dimensions가 없으면 0으로 채우세요.

- summaryText는 비어 있으면 다음 문장으로 채우세요:
  "전체적인 평가를 생성하는 데 일부 정보가 부족했지만, 주어진 내용을 바탕으로 정리했습니다."

- summaryText는 반드시 문자열이어야 하며 null, 객체, 배열은 허용하지 않습니다.

- strengths는 문자열 배열이어야 합니다.
  - null이면 빈 배열 []로 변환하세요.
  - 배열이 아니면 배열로 감싸세요.
  - 문자열이 아닌 값은 제거하세요.

- improvements는 문자열 배열이어야 합니다.
  - null이면 빈 배열 []로 변환하세요.
  - 배열이 아니면 배열로 감싸세요.
  - 문자열이 아닌 값은 제거하세요.

- 모든 필드는 반드시 포함되어야 합니다.
- null 값은 절대 허용하지 않습니다.
- 객체나 중첩 구조는 dimensions를 제외하고 허용하지 않습니다.

- 값이 잘못된 경우:
  - 숫자가 아닌 값 → 0으로 변환
  - 범위를 벗어나면 0~100 범위로 클램핑

출력은 반드시 유효한 JSON 객체 하나만 반환하세요.
