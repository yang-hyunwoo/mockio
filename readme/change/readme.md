<summary> 수정 사항 (2026.04.01)</summary>

#### 📌 문제 상황
- 꼬리 질문 생성 시 의미 없는 답변 , 짧은 답변 일 경우 무조건 꼬리 질문을 생성하는 문제가 있었습니다.
- 딥 다이브 질문 생성 시 의미 없는 답변 ,짧은 답변 애매한 답변 일 경우 딥 다이브 질문을 생성하는 문제가 있었습니다.

#### 🔍 원인
- 꼬리 질문 생성 시 서버에서 체크 [빈값 짧은 답변 , 금지어] 만 하고 있었습니다.
 그래서 빈값 , 금지어 제외 하고 의미 없는 답변 , 짧은 답변 일 경우 답변이 이상해도 꼬리질문을 생성하고 있었습니다.
- 딥다이브 질문 생성 서버에서 체크 [빈값 , 짧은 답변 , 금지어] 만 하고 있었습니다.
 그래서 빈값 , 금지어 , 짧은 답변을 제외 하고 답변이 질문과 상관 없어도 딥다이브 질문을 생성 하고 있었습니다.

#### 🛠 해결
- 꼬리 질문 생성 시 서버에서 체크 [빈값] 을 하고
  꼬리질문을 생성할 수 있는지에 대한 ai api를 호출 하고 응답 값이 true 일 경우에만
  꼬리질문 생성 ai api를 호출하여 꼬리질문을 내려 주게 수정을 하였습니다.
- 딥다이브 질문 생성 시 서버에서 체크 [빈값] 을 하고
  딥다이브 질문을 생성할 수 있는지에 대한 ai api를 호출 하고 응답 값이 true 일 경우에만
  딥다이브 질문 생성 ai api를 호출하여 딥다이브 질문을 내려 주게 수정을 하였습니다.

<summary> 수정 사항 (2026.04.06)</summary>

#### 📌 문제 상황
- 부하 테스트 진행 중 로그인 API에서 응답 시간이 평균 2초 이상으로 나타나며, 일부 요청은 p95 기준 최대 8초까지 지연되는 문제가 발생했습니다.

#### 🔍 원인
- 로그인 성공 후 auth-service에서 core-service로 전달하는 API 호출이 동기 방식으로 처리되고 있었습니다.
- 해당 호출은 로그인 핵심 로직과 무관한 부가 작업(실패 카운트 초기화, 마지막 로그인 시간 갱신)이었음에도 응답 반환 전에 실행되어 전체 latency를 증가시키는 원인이 되었습니다.
- 특히 동시 요청 증가 시 외부 API 응답 대기로 인해 스레드가 점유되며 tail latency가 크게 증가했습니다.

#### 🛠 해결
- 로그인 핵심 경로와 부가 작업을 분리하기 위해, 해당 로직을 Kafka 기반 비동기 이벤트 처리 방식으로 변경했습니다.
- 로그인 성공 시 이벤트를 발행하고, core-service에서 이를 소비하여 후처리를 수행하도록 구조를 개선했습니다.

#### 📊 개선 결과
- median: 약 2.5s → 1.46s
- p95: 약 8s → 2.48s

동기 호출 제거를 통해 응답 지연을 줄이고, tail latency를 크게 개선할 수 있었습니다.
